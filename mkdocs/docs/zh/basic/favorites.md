# 收藏与导入

收藏功能适合做这几件事：

- 把已经验证过的提示词单独沉淀下来
- 给提示词补标题、说明、分类和标签
- 在不同工作区之间快速复用
- 单独导入导出一组收藏，而不动整包数据

它不是历史记录的别名。

## 最简单的理解

- **历史记录**：更像版本链和工作过程
- **收藏**：更像你主动挑出来的可复用成果

## 收藏入口在哪里

收藏相关入口主要有两类：

### 顶部工具栏里的收藏管理

这里适合：

- 搜索
- 筛选
- 浏览全部收藏
- 导入导出
- 管理分类和标签

### 工作区或结果区域里的“保存收藏”

这里适合把当前这条内容直接沉淀下来。

保存时你可以补充：

- 标题
- 描述
- 分类
- 标签
- 所属模式
- 对于图片收藏，还可以附带图片并设置封面

## 收藏管理器当前能做什么

根据当前实现，可以确认收藏管理器支持：

- 关键词搜索
- 按分类筛选
- 按标签筛选
- 网格卡片浏览
- 预览收藏内容
- 复制内容
- 立即使用
- 编辑
- 删除
- 导入 JSON
- 导出 JSON
- 清空全部收藏
- 分类管理
- 标签管理

## “立即使用”会发生什么

收藏不是简单复制一段文本。

当前点击 **立即使用** 时，系统会尽量根据收藏里的模式信息做回填：

- 文本收藏会切到对应的 `basic` 或 `pro` 工作区
- 如果收藏记录了 `system / user` 语义，也会尽量切到对应子模式
- 图片收藏会切到 `文生图`、`图生图` 或 `多图生图`

然后再把收藏内容回填到对应工作区。

所以收藏更接近“带工作区语义的复用入口”，而不只是剪贴板。

## 收藏和历史记录该怎么分工

一个很实用的工作流是：

1. 先在左侧工作区持续优化
2. 用历史记录保留过程版本
3. 当某个结果已经足够稳定时，再保存到收藏
4. 后续优先从收藏里复用，历史记录只在需要追溯过程时再看

## 收藏和数据管理的区别

### 用收藏导入导出

适合：

- 只迁移一部分精选内容
- 分享给别人一组提示词
- 保持主环境数据尽量干净

### 用数据管理导入导出

适合：

- 整机备份
- 整体迁移
- 连模型、模板、历史一起搬走

## 还可以附带哪些信息

保存收藏对话框当前支持：

- 标题
- 描述
- 分类
- 标签
- 功能模式
- 文本模式下的优化模式
- 图像模式下的子模式
- 正文内容编辑
- 图片上传
- 设置封面图
- 删除已附带图片

这意味着收藏不仅能保存文本，还能保存一组和提示词相关的媒体信息。

如果这条收藏原本来自 `多图生图`，`立即使用` 也会尽量恢复对应的多图子模式和媒体上下文，避免你还要手动重新整理这组图片。

## 关于 Prompt Garden 的说明

当前实现里，收藏预览支持可插拔扩展。

如果部署时启用了 `Prompt Garden` 相关集成，收藏预览区还可能显示额外的外部快照信息和媒体内容。

这属于可选集成能力，不是所有部署都会默认开启。

## 使用建议

- 历史记录留过程，收藏留结果
- 标题尽量写成未来还能看懂的名字，不要只写“测试1”
- 标签不要太多，能帮助筛选就够
- 图片收藏建议明确设置封面，否则后面浏览会不够直观
- 分享收藏 JSON 前，最好先检查说明和媒体内容里有没有敏感信息
