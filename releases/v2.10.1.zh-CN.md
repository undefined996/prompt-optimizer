# Prompt Optimizer v2.10.1

## 概括
- 修复英文文档中残留的中文词汇（replicate、smart-fill、style-learn 页面）。
- 补充 v2.10.0 功能文档，覆盖收藏示例、模型配置详情、Garden 用法、并行测试和桌面端直连。
- 通过 provide/inject 模式恢复 Pro 工作区的变量管理器事件处理。

## 亮点
本次补丁版本提升了 v2.10.0 功能集的文档质量，并修复了 Pro 工作区中变量管理器无法打开的问题。

## 产品更新
### Web
- 更新落地页文案，从"5大场景"调整为"4类工作区"。
- 移除场景覆盖区域中独立的 Prompt Garden 和智能收藏卡片。

### Core/Infra
- 新增 provider 请求风格说明（Chat Completions vs Responses）和 DeepSeek 思考参数文档。
- 新增图像模型能力标签和版本特定预设说明（如 Seedream 4.5 与 5.0 Lite 的区别）。
- 新增桌面端 localhost/私有网段直连路由说明。
- 新增测试评估模式"全部运行"并行执行说明。
- 新增收藏示例应用到工作区的说明。

## 修复
- 修复英文文档中残留的中文词汇（replicate.md、smart-fill.md、style-learn.md）。
- 修复 index 页面中 Prompt Garden 链接路径，从 `/basic/garden` 改为 `/basic/prompt-garden`。
- 修复 mkdocs-dev.yml 导航，将 optimization 拆分为系统提示词和用户提示词工作区页面。
- 修复 ContextUserWorkspace.vue 中 `@open-variable-manager` 事件未触发的问题，添加 provide/inject 处理。
- 移除 ContextUserWorkspace.vue 中未使用的 `open-tool-manager` emit。
- 解决 P1/P2/P3 review 发现的问题：移除未使用的 inject、修复 saveToGlobal 绕过持久化、移除尾部空白。

## 破坏性变更 / 升级说明
- 无。

## 开发者说明
- `openToolManager` emit 已从 ContextUserWorkspace.vue 中移除，工具管理现在通过其他机制处理。
- 变量管理器打开现在使用 provide/inject 模式并带有 emit 回退，与 Pro 工作区的其他操作保持一致。
- 英文文档已进行中文残留审查，未来的文档 PR 应检查是否存在中英混杂内容。
