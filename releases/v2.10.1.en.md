# Prompt Optimizer v2.10.1

## Summary
- Fixes residual Chinese text in English documentation pages for auxiliary features (replicate, smart-fill, style-learn).
- Adds v2.10.0 feature documentation covering favorites examples, model provider details, Prompt Garden usage, parallel testing, and desktop direct routing.
- Restores variable manager event handling in Pro workspace via provide/inject pattern.

## Highlights
This patch release improves documentation quality for the v2.10.0 feature set and fixes a broken variable manager action in the Pro workspace.

## Product Updates
### Web
- Updated landing page copy to reflect 4 workspace types instead of 5 scenarios.
- Removed standalone Prompt Garden and Smart Favorites scenario cards from the coverage section.

### Core/Infra
- Added documentation for provider-specific request details (Chat Completions vs Responses style) and DeepSeek thinking parameters.
- Added documentation for image model capability tags and version-specific presets (e.g., Seedream 4.5 vs 5.0 Lite).
- Added documentation for localhost/private network direct routing in desktop builds.
- Added documentation for Run All parallel execution in testing/evaluation mode.
- Added documentation for favorites example application to workspaces.

## Fixes
- Fixed Chinese text残留 in English documentation pages (replicate.md, smart-fill.md, style-learn.md).
- Fixed Prompt Garden link path from `/basic/garden` to `/basic/prompt-garden` in index pages.
- Fixed mkdocs-dev.yml navigation to split optimization into system and user workspace pages.
- Fixed `@open-variable-manager` event not firing in ContextUserWorkspace.vue by adding provide/inject handler.
- Removed unused `open-tool-manager` emit from ContextUserWorkspace.vue.
- Resolved P1/P2/P3 review findings: removed unused injects, fixed saveToGlobal bypassing persistence, removed trailing whitespace.

## Breaking Changes / Upgrade Notes
- None.

## Developer Notes
- The `openToolManager` emit has been removed from ContextUserWorkspace.vue. Tool management is now handled through a different mechanism.
- Variable manager opening now uses provide/inject pattern with emit fallback, consistent with other Pro workspace actions.
- English documentation has been audited for Chinese text残留. Future documentation PRs should check for mixed-language content.
